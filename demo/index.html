Jughandle Tracker

Automatic Referral Tracking

Inspired by @mattangriffel

Original Idea Post

This one is kind of technical, but every page on a product should append a unique parameter (something like ?id=532391) that way when I copy the URL and send it to a friend, they know who referred who. And a link could be permanently created between users

Current Working Example

Add 1 line of JS to your website (or customize the param value and cookie, so 3 lines). Once added, you can then pass in a param and it will automatically keep that param as a visitor goes from page to page. It uses both cookies and localstorage, to keep track of the param, so you could remove one or the other and still be tracked. Take a look by clicking the example link below and then going to the About or Contact page.

Example Track Link

Now visit the About and Contact Page, notice the jhid is automatically appended without any code changes to the link!

How It Works?

Jughandle uses cookies and localstorage to save the ID being tracked and automatically add it to a URL when loaded

Why?

A simple way to track visitors and referral without a huge/expensive system.Right now this is just a tracker, but planning to build a MVP product. For now, you have to keep track of the IDs you set e.g. Track index.html?jhid=123 where 123 represents Acme Company.

What about X feature

This is still a work in progress, so feel free to reach out @johnmurch or on GitHub

What about a SAAS?

I built Jughandle as a proof of concept with the intention of building a SAAS to track visitors and who they forward links to

Where Should I use this?

Jughandle works on any website, but right now it's a bit manual. You have to manually keep track of an ids for each visitor or community you want to track.Be sure to add jh_track (or your customized name) as a param to ignore in Google Analytics
